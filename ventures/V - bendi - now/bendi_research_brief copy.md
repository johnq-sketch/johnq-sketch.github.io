## WS-1 · 市场研究组

**负责人**：市场研究员（1人）
**核心问题**：这个市场足够大吗？需求是真实的吗？时机对吗？

---

### 任务 1.1 · TAM / SAM / SOM 建模

**要回答的问题**：
- 中国在线体验预订市场的全量规模是多少？
- 其中"文化遗产城市 + 非标深度体验"这个细分市场有多大？
- 「本地」在 3 年内实际能吃到多少？

**研究方法**：
- **自上而下**：从文旅总消费规模 → 在线化比例 → 体验品类占比 → 非标深度体验占比，逐层拆解
- **自下而上**：以大同为基准，测算单城市天花板，再乘以目标城市数量
- **对标法**：找到 OTC、Airbnb Experiences、Klook 的公开 GMV 数据，反推市场规模

**数据来源清单**：
- 文旅部《中国旅游统计年鉴》
- 艾瑞咨询《中国在线旅游行业报告》
- 大同市文旅局官方数据（需要电话或实地获取）
- OTC 小镇公众号/媒体报道中的数据点
- Airbnb 年报中 Experiences 板块数据

**交付物**：
- TAM/SAM/SOM 三层数字（含假设说明）
- 大同单城市 GMV 天花板测算表（Excel）
- 一张可以放进 BP 的市场规模可视化图

---

### 任务 1.2 · 市场时机分析（Why Now）

**要回答的问题**：
- 为什么这个市场在现在是窗口期，而不是 3 年前或 3 年后？
- 有哪些宏观/政策/技术/文化趋势正在支撑这个需求？

**研究方法**：
- 搜集过去 3 年"在地体验"相关关键词在小红书、微信指数的搜索量趋势
- 整理文旅部、发改委关于"沉浸式体验"、"文化遗产活化"的政策文件清单
- 《黑神话：悟空》上线后大同旅游数据变化（携程/飞猪公开数据）
- Z 世代旅游消费行为变化的第三方报告

**交付物**：
- "Why Now"论据清单（至少 5 条，每条有数据支撑）
- 政策利好时间线

---

### 任务 1.3 · 需求验证调研

**要回答的问题**：
- 旅行者真的愿意为这类体验付钱吗？在什么价格区间？
- 他们的实际决策路径是什么？

**研究方法**：
- **问卷调查**：设计 15 题问卷，通过小红书旅行博主合作分发，目标样本 200 人
  - 核心问题：旅行中是否参与过非景区体验？花了多少？满意度如何？
  - 关键问题：给出 5 个体验描述（含价格），哪个最愿意购买？
- **小红书数据挖掘**：抓取"大同 旅行"、"大同 体验"、"大同 值得做"相关笔记的评论区，分析真实需求语言

**问卷核心设计要点**（给研究员的指令）：
- 必须包含**价格锚定题**：给出具体体验描述，问"¥198 / ¥298 / ¥398 你会购买吗"
- 必须包含**渠道信任题**：你更倾向在哪个渠道预订此类体验（选项含小红书私聊/微信群/专属平台/OTA）
- 必须包含**购买障碍题**：是什么让你没有购买过此类体验

**交付物**：
- 问卷完整版（可直接投放）
- 问卷数据分析报告（含图表）
- "用户真实语言库"——从评论区收集的 50 条真实用户原话，供 BP 文案使用

## WS-2 · 用户研究组

**负责人**：用户研究员（1人）
**核心问题**：我们的用户是谁？他们的真实痛点和潜在需求是什么？

---

### 任务 2.1 · C 端用户深度访谈

**访谈目标**：12–15 人，覆盖三类 Persona

**招募标准**：
- 过去 12 个月内有过文化遗产城市旅行经历
- 年龄 25–42 岁，一二线城市居民
- 至少 5 人曾经通过小红书/微信私聊预订过非标体验

**访谈大纲**：

```
模块一：旅行行为（10 分钟）
- 讲一次你觉得最满意的旅行，是什么让它特别？
- 你通常怎么规划一次旅行中"要做什么"？

模块二：体验消费（15 分钟）
- 你有没有在旅行中做过"非景区"的事？讲一讲
- 如果有过小红书/微信私聊预订，整个过程是怎样的？
  ↳ 追问：最麻烦的是哪一步？有没有出过问题？

模块三：痛点挖掘（15 分钟）
- 有没有你"想做但最终没有做到"的体验？为什么没做到？
- 如果给你一个平台，能让你直接预订"跟本地人一起做一件事"，
  你最担心的是什么？（不引导，让对方自由回答）

模块四：价值评估（10 分钟）
- 展示 3 个具体体验产品描述（含定价），询问：
  "你会购买哪个？为什么？"
  "如果你买了，你会跟谁说？怎么说？"
```

**特别关注**：
- **潜在需求（Latent Demand）**——用户说不出来但行为暴露的需求。例如：用户说"想了解本地文化"，但追问下发现其实是"想有一个可以分享的故事"——这是两个完全不同的产品方向。

**交付物**：
- 访谈录音 + 转录文本（全部 12–15 份）
- 用户洞察报告：3 个核心 Persona（含引用原话）
- 痛点层级图：表层痛点 vs 深层动机 vs 潜在需求的三层拆解
- "用户愿意付钱的真实理由"清单（BP 最重要的论据之一）

---

### 任务 2.2 · 传统渠道的痛点地图

**要回答的问题**：
- 用户现在是怎么完成"找到 + 预订 + 参与"这个全链路的？
- 每一个环节的摩擦点在哪里？

**研究方法**：
- **用户旅程重建**：让 5 位访谈用户在访谈中实操演示"我是怎么在小红书上找到并预订一个体验的"，全程录屏
- **渠道体验测试**：研究员自己以用户身份，分别在小红书私聊、微信群、OTA 平台各完成一次非标体验预订，记录每个步骤的时间和摩擦

**交付物**：
- 现有渠道用户旅程图（标注每步骤的情绪曲线和摩擦点）
- "小红书/微信群预订体验"的完整体验报告（含截图，可直接放进 BP）
- 用一句话量化痛点：例如"完成一次非标体验预订平均需要 X 次沟通、跨 X 个平台"

## WS-3 · 竞品研究组

**负责人**：竞品分析员（1人）
**核心问题**：竞争格局是什么？我们的差异化能否站住脚？

---

### 任务 3.1 · 全竞品地图

**需要覆盖的竞品清单**：

| 类别 | 竞品 | 研究优先级 |
|------|------|-----------|
| 全球在线体验平台 | Airbnb Experiences | ★★★★★ |
| 全球体验预订 | Viator、GetYourGuide、TripAdvisor 体验 | ★★★★ |
| 亚洲体验预订 | Klook、KKday | ★★★★★ |
| 国内 OTA 体验板块 | 携程玩乐、飞猪体验、美团玩乐 | ★★★★★ |
| 国内垂直体验平台 | OTC 小镇、小岛游、黄包车、纯粹 | ★★★★★ |
| 内容平台（兼具交易） | 小红书（私域预订）、抖音本地生活 | ★★★★ |
| 社群渠道 | 微信群（各类旅行/文化社群） | ★★★ |
| 本地向导类 | 各类地陪平台（当地向导、地陪网） | ★★★ |

**每个竞品需要研究的维度**：

```
1. 商业模式
   - 谁付钱？平台如何赚钱？抽佣比例？
   - B 端还是 C 端驱动？

2. 产品形态
   - 体验如何上架？谁来审核？
   - 用户体验流程（含截图）
   - 定价区间

3. 供给侧策略
   - 如何招募 Host/供应商？
   - 有没有质量控制机制？

4. 需求侧策略
   - 主要获客渠道？
   - 用户评价机制？

5. 在中国市场的表现
   - 月活/GMV（能找到的公开数据）
   - 在中国的失败/受限原因（尤其是 Airbnb/Klook）

6. 在"北方文化遗产城市"的供给密度
   - 在大同/西安/洛阳上架了多少体验？质量如何？
   - （直接去每个平台搜索"大同 体验"，截图记录结果数量和定价）
```

**交付物**：
- 竞品对比矩阵（Excel，覆盖上述全部维度）
- 每个一级竞品的 1 页简报（含截图、核心数据、优劣势）
- **Airbnb Experiences 在中国失败的深度复盘**（投资人必问问题）
- **Klook / 携程玩乐在非标体验上的结构性缺陷分析**
- 竞争格局四象限图（可直接放进 BP）

### 任务 3.2 · 黄包车 & 纯粹 深度研究

这两个是最接近「本地」定位的国内竞品，需要单独深度研究。

**研究问题**：
- 它们的产品模式是什么？靠什么盈利？
- 它们在哪些城市？覆盖什么体验品类？
- 它们的 Host 招募和管理机制是什么？
- 它们有什么做得好的？有什么明显的问题？
- 它们在北方文化遗产城市有没有布局？

**研究方法**：
- 直接下载/使用这两个平台，完整体验用户旅程
- 找到它们的媒体报道/创始人访谈（36氪、虎嗅等）
- 若可能，以"潜在合作方"名义联系其商务，了解 Host 合作条款

**交付物**：
- 黄包车 vs 纯粹 vs 「本地」三方对比简报
- 「本地」相对这两个平台的**可辩护差异化**清单（具体的、可验证的差异，不是模糊的"更好"）

## WS-4 · 供给侧研究组

**负责人**：供给侧研究员（1人，需要能够进行实地研究）
**核心问题**：Host 供给是否真实存在且可规模化？

---

### 任务 4.1 · 大同 Host 资源实地调研

**目标**：建立"大同可转化 Host 候选名录"，每个候选包含：

```
基本信息：姓名/机构、联系方式、所在位置

体验资产评估：
  - 核心技能/场景是什么？
  - 可开放的体验时长和频次？
  - 独特性评分（1–5）
  - 参与感评分（1–5）
  - 叙事性评分（1–5）

合作意愿评估：
  - 是否有意愿合作（初步接触后判断）
  - 主要顾虑是什么？
  - 潜在的"入驻触发条件"是什么？
```

**调研渠道**：
- 东南邑艺术街区（实地走访，与运营团队和入驻主理人逐一接触）
- 云冈石窟研究院文保部门（已有合作基础，确认合作意愿和具体条款）
- 平城学堂（确认人文社群资源的可转化性）
- 山西省专家学者协会（了解其网络的实际覆盖范围）
- 大同本地独立咖啡馆（实地拜访，寻找有故事的主理人）
- 大同本地小红书达人（筛选"在地内容创作者"，评估 Host 潜力）

**交付物**：
- 大同 Host 候选名录（至少 20 个候选，含评分）
- 前 10 个优先级 Host 的详细 profile
- Host 供给侧"冷启动可行性报告"：3 个月内能否完成 5 个体验产品上线？

---

### 任务 4.2 · Host 心理模型研究

**要回答的问题**：
- Host 的真实决策障碍是什么？（不是我们假设的，而是他们说出来的）
- 什么样的条件会让他们说"好，我试试"？

**研究方法**：
- 对 10 位潜在 Host 进行深度访谈（30–45 分钟）
- **访谈框架**：

```
1. 你现在收入来源是什么？有没有想过做旅行相关的事？
2. 你有没有听说过 Airbnb 体验 / OTC 小镇？你怎么看？
3. 如果有人告诉你"你可以把你的 X 变成一个旅行者愿意付钱的体验"，
   你第一反应是什么？最担心什么？
4. 你需要哪些支持，才会愿意试试？
```

- 与已经在 OTC 小镇或其他平台做过体验的 Host 对话（了解其真实收益和运营负担）

**交付物**：
- Host 心理模型报告：核心顾虑的优先级排序，以及每个顾虑的具体表现
- "Host 入驻的临界条件"清单（BP 供给侧策略的核心依据）

---

### 任务 4.3 · 酒店合作可行性研究

**要回答的问题**：
- 大同有哪些精品酒店/民宿适合作为渠道合作方？
- 酒店前台推荐体验的实际执行意愿有多强？障碍是什么？
- 最有效的合作利益结构是什么？

**研究方法**：
- 梳理大同评分 4.5 分以上精品民宿/酒店名单（小红书、携程筛选）
- 以"潜在合作方"身份实地拜访 5–8 家，与前台或运营负责人交流
- **访谈核心问题**：
  - 住客最常问的"周边体验"问题是什么？
  - 你们现在是怎么回答的？
  - 如果有一张精选体验菜单，你们愿意放在前台吗？顾虑是什么？
  - 什么样的分成方式会让你觉得值得推？

**交付物**：
- 大同精品酒店/民宿名单（含联系人）
- 酒店合作意愿评估报告
- 最优酒店合作利益结构建议（直接输入 BP 第 7 章）

## WS-5 · 商业模型组

**负责人**：商业分析师（1人，需要具备财务建模能力）
**核心问题**：这个生意的钱从哪来？能赚多少？什么时候开始赚？

---

### 任务 5.1 · 单体经济模型（Unit Economics）

**建模要求**：

```
Layer 1：单场体验经济
- 输入变量：客单价、人数上限、入座率、平台抽佣率
- 成本项：支付手续费、保险、运营分摊
- 输出：单场平台净利润、盈亏平衡入座率

Layer 2：单用户经济
- 输入变量：获客成本（CAC）、首单转化率、复购率、客单价
- 输出：LTV、LTV/CAC 比率、回收周期

Layer 3：单 Host 经济（站在 Host 视角）
- Host 每月最低收入需求是多少才愿意留存？
- 在不同抽佣比例下，Host 的实际到手收益是多少？
- 设计：让 Host 觉得"比自己在小红书卖更划算"的分成结构
```

**敏感性分析要求**：
- 对以下变量做敏感性测试：客单价 ±20%、入座率 40%/60%/80%、抽佣比例 10%/15%/20%
- 输出一张"在哪些条件组合下，平台能在 12 个月内达到现金流平衡"的热力图

**交付物**：
- 单体经济模型 Excel（含所有假设，可实时调参）
- 盈亏平衡分析（月均需要多少场次才能覆盖固定成本）
- Host 分成结构建议（基于 Host 留存意愿和平台盈利的双重约束）

---

### 任务 5.2 · 18 个月财务预测

**建模框架**：

```
Phase 1（M1–M6）大同验证期
  - 假设：体验数量成长曲线、月订单量曲线、固定成本
  - 输出：累计 GMV、累计亏损、所需启动资金

Phase 2（M7–M18）城市扩展期
  - 假设：新城市复制速度、边际运营成本增量
  - 输出：现金流图、盈利预测时间点

融资情景分析
  - 不融资：自有资金能走多远？
  - Pre-seed ¥150 万：能支撑到什么里程碑？
  - Pre-seed ¥250 万：如何加速？
```

**交付物**：
- 18 个月财务模型 Excel（三情景：保守/基础/乐观）
- 现金流图（可直接放进 BP）
- 融资时机与金额建议（基于模型，不是拍脑袋）

---

### 任务 5.3 · 商业模式完整性验证

**要回答的问题**：
- 我们的收入是否依赖单一来源？各收入层的占比和稳定性如何？
- 有没有更好的商业模式选择（如订阅制、会员制、B 端数据服务）？

**研究方法**：
- 对标研究：Airbnb Experiences、Klook、OTC 的收入结构演化历史
- 构建"收入多元化路线图"：C 端预订 → 酒店渠道分成 → 企业团建 → B 端数据接口，每个层次的触发条件是什么

**交付物**：
- 商业模式完整性分析报告
- 收入多元化路线图（含时间节点和触发条件）

## WS-6 · 增长与 GTM 组

**负责人**：增长策略师（1人）
**核心问题**：第一批用户怎么来？增长引擎是什么？

---

### 任务 6.1 · 冷启动流量策略

**要回答的问题**：
- 在零广告预算的情况下，第一个月的 20 个用户从哪里来？
- 内容驱动的增长飞轮具体怎么运转？

**研究要求**：

**小红书渠道深度研究**：
- 分析"在地体验"相关账号的内容策略（选取 10 个账号分析）
- 研究：什么样的内容在小红书上带来体验类产品的实际转化？

### 任务 7.1 · 风险登记册

**研究要求**：

对以下风险维度，每一项都需要给出：
- **风险描述**（具体，不能模糊）
- **发生概率**（高/中/低，需要论据）
- **影响程度**（高/中/低）
- **早期预警信号**（什么数据或事件出现时说明风险正在发生）
- **应对预案**（具体动作，不能只说"我们会应对"）

**风险分类**：

```
业务风险
├── 需求侧：用户付费意愿不足
├── 供给侧：Host 流失或质量下降
├── 渠道风险：微信小程序政策变化
└── 季节性：旅游淡旺季导致现金流断裂

竞争风险
├── OTC / 携程进入北方市场
├── 小红书推出内置预订闭环
└── 大型资本扶持的新竞品出现

执行风险
├── 创始团队分裂
├── 关键 Host 被竞品挖走
└── 技术开发延期

法务与监管风险
├── 导游/旅游服务资质要求（Host 是否需要持证？）
├── 平台支付牌照要求（MoR 模式的合规性）
└── 文化遗产相关活动的政策限制
```

**法务研究重点**：
- 在中国，个人提供旅游体验服务是否需要导游资质？相关法规如何界定？
- 微信支付代收代付（MoR 模式）对平台的资质要求是什么？
- 在云冈石窟等文保单位内/周边开展商业体验活动的审批要求？

**交付物**：
- 完整风险登记册（Excel，含优先级排序）
- 法务合规要点清单（含建议的应对方式）
- Phase 1 最需要关注的前 5 大风险的详细应对预案

## 架构师的质量控制标准

每个工作组提交报告时，用以下标准审核，不达标的退回重做：

1. **每个结论都必须有数据来源**——不接受"普遍认为"、"通常情况下"、"有数据显示"这类模糊表达
2. **数字要有置信区间**——不能只给一个点估计，要给范围和假设说明
3. **竞品分析必须是第一手的**——必须亲自使用过这个产品，不能只靠二手报道
4. **用户洞察必须有原话支撑**——每个 Persona 结论至少要有 2 条用户原话作为证据
5. **风险必须有预警信号和具体预案**——不能只描述风险，要告诉我看到什么信号要触发什么动作
